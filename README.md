# WEB2026 — Space Oddity 🚀

A flexbox gym. Seven small layout jobs, from *fix the bug* to *build the whole thing from scratch*. By the end, `display: flex` lives in your fingers.

Flexbox is the art of handing a row (or a column) of boxes some space and saying "share it sensibly." That's it. That's the whole idea. These exercises just drill it until it's automatic.

## Getting started

1. **Fork** this repository, then clone your fork with GitHub Desktop
2. Open the folder in VS Code
3. Install the **Live Server** extension (Extensions view, search "Live Server" by Ritwick Dey)
4. Right-click `index.html` and select **"Open with Live Server"**

The landing page (`index.html`) links every exercise. Do them in order — each one assumes the one before.

## The rules

- **Filenames are always lowercase.** Always. (You'll learn *why* on Day 4.)
- When you're stuck for more than a few minutes, run the **Stuck Protocol**: read the error out loud → rubber-duck it → check the docs → ask a neighbor → ask me or an LLM with the actual code.
- Froggy is a legitimate solving strategy. [Flexbox Froggy](https://flexboxfroggy.com) levels 1–13 cover everything in here.
